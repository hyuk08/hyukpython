# Python 가상환경 설정 가이드

이 프로젝트의 라이브러리 충돌을 방지하기 위해 가상환경을 사용합니다.

## 방법 1: pipenv 사용 (권장)

프로젝트에 `Pipfile`이 있으므로 pipenv를 사용하는 것을 권장합니다.

### 1. pipenv 설치
```bash
pip install pipenv
```

### 2. 가상환경 생성 및 패키지 설치
```bash
pipenv install
```

### 3. 가상환경 활성화
```bash
pipenv shell
```

### 4. 가상환경에서 Python 실행
```bash
# 가상환경 활성화 후
python item/app.py

# 또는 가상환경 활성화 없이 직접 실행
pipenv run python item/app.py
```

### 5. 새 패키지 설치
```bash
pipenv install 패키지명
```

### 6. 가상환경 비활성화
```bash
exit
```

## 방법 2: venv 사용 (표준 Python 가상환경)

### 1. 가상환경 생성
```bash
python3 -m venv venv
```

### 2. 가상환경 활성화
```bash
# macOS/Linux
source venv/bin/activate

# Windows
venv\Scripts\activate
```

### 3. 패키지 설치
```bash
pip install -r requirements.txt
```

### 4. 가상환경 비활성화
```bash
deactivate
```

## 문제 해결

### 가상환경이 활성화되지 않을 때
```bash
# pipenv 사용 시
pipenv --rm  # 기존 환경 삭제
pipenv install  # 새로 생성

# venv 사용 시
rm -rf venv  # 기존 환경 삭제
python3 -m venv venv  # 새로 생성
```

### 패키지 버전 충돌 시
```bash
# pipenv 사용 시
pipenv install --dev  # 개발 패키지 포함 설치

# venv 사용 시
pip install --upgrade 패키지명
```
